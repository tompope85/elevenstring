# Contributing to elevenstring

Contributions — corrections, adjacent-work references, and proposals — are welcome.

## Developer Certificate of Origin (DCO)

To keep the project's rights clear and preserve its licensing options, every contribution must be made under the **Developer Certificate of Origin** (see [DCO](./DCO)). The DCO is a lightweight, widely-used alternative to a formal Contributor License Agreement: by signing off, you certify that you wrote the contribution (or otherwise have the right to submit it) and that it may be distributed under the project's licenses.

### How to sign off

Add a `Signed-off-by` line to each commit, using your real name and email:

```
Signed-off-by: Your Name <you@example.com>
```

You can do this automatically with:

```
git commit -s
```

Commits without a sign-off cannot be merged.

## What you're agreeing to

By signing off you agree that:

- Documentation/specification contributions are licensed under **CC BY-SA 4.0**.
- Software contributions are licensed under **AGPL-3.0**.
- You grant the author the rights described in the DCO, which preserves the project's ability to offer a separately-licensed edition in future.

See [LICENSING.md](./LICENSING.md) for the full licensing picture.

## Scope and style

- Keep changes focused and explain the reasoning in the commit message.
- For substantive framework proposals, open an issue first — per the framework's own logic, an extension is a *typed move with provenance, evidence, and impact analysis*.
